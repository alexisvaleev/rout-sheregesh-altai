## О проекте

**«Две легенды»** — туристическое приложение, которое объединяет два культовых направления Сибири: горнолыжный курорт **Шерегеш** и **Горную Шорию** с одной стороны, и **Республику Алтай** с другой. Приложение предлагает 6 готовых маршрутов разной сложности и длительности, от коротких уикенд-поездок до полноценных экспедиций на 7 дней.

Ключевые особенности:
- Интерактивная карта с отрисовкой реальных автомобильных маршрутов через OSRM
- Фотогалереи с каруселью для каждого маршрута
- Система тарифов (Базовый / Комфорт / Премиум) с детальным списком включённых услуг
- Геймификация: 8 уровней пользователей и 25 достижений
- Система отзывов с рейтингом
- Тёмная и светлая темы с сохранением выбора
- Аутентификация по телефону (симуляция)
- Вишлист (избранные маршруты)
- Виджет погоды через OpenWeatherMap (демо-режим)
- Индикатор сезона с календарём лучших месяцев для поездки

## Функциональность

### Основные возможности

- **Главный экран:** карта всех маршрутов + список с фильтрацией по сезону и сортировкой
- **Детальная страница маршрута:** карта, галерея, описание, точки интереса, тарифы, отзывы, погода
- **Фотогалерея:** горизонтальная карусель на странице маршрута + полноэкранная галерея всех фото в профиле
- **Система сбора фотографий:** авторизованные пользователи могут собирать фото в каждой точке маршрута; прогресс отображается в галерее и в профиле; за 15 собранных фото — достижение «Фотомастер»
- **Интерактивная карта Leaflet:** OpenStreetMap, OSRM-маршруты, кластеризованные маркеры POI с эмодзи
- **Геймификация:** 8 уровней пользователей, 25 достижений (очки за пройденные маршруты, собранные фото, избранное, дистанцию), прогресс-бар уровня
- **Тёмная тема:** переключение в профиле, сохраняется в localStorage, защита от вспышки при загрузке
- **Аутентификация:** по телефону (+7 маска) — симуляция, тестовый профиль со всеми достижениями
- **Система отзывов:** рейтинг + текстовый отзыв для каждого маршрута
- **Избранное:** добавление маршрутов в вишлист (только для авторизованных)
- **Виджет погоды:** OpenWeatherMap (демо-режим) с отображением температуры, ощущений, влажности, ветра

### Состояния UI

- **Загрузка:** спиннеры при инициализации карты, загрузке погоды, отправке кода авторизации
- **Пусто:** заглушки для отсутствующих отзывов, маршрутов в профиле, избранного
- **Ошибка:** обработка ошибок карты (OSRM), загрузки погоды; страница 404 для несуществующих маршрутов
- **Офлайн:** виджет погоды обрабатывает сетевые ошибки и показывает заглушку

## Установка и запуск

### Требования

- **Node.js** >= 18
- **npm** или **yarn**
- **Expo CLI** (`npx expo` используется в скриптах)
- Для iOS: macOS + Xcode
- Для Android: Android Studio + Android SDK
- Для физического устройства: приложение **Expo Go** (iOS/Android)

### Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone <url>
cd route-app

# 2. Установить зависимости
npm install

# 3. Запустить веб-версию (откроется http://localhost:8081)
npm run web

# iOS (требуется macOS + Xcode)
npm run ios

# Android (требуется Android SDK)
npm run android

# Универсальный запуск с QR-кодом
npx expo start
```

### Запуск на телефоне

```bash
# Через Expo Go (сканировать QR-код)
npx expo start

# Туннель (если телефон в другой сети)
npx expo start --tunnel
```

## Архитектурные заметки

### Темизация

- Все стили используют фабричную функцию `getStyles(C: ThemeColors)` — компонент получает объект цветов и строит `StyleSheet.create()`.
- Цвета подставляются на лету (не через `useTheme` в стилях, а пересозданием styles при смене темы: `useMemo(() => getStyles(Colors), [Colors])`).
- Тема синхронно инициализируется из `localStorage` до первого рендера, чтобы избежать белой вспышки на вебе.

### Карта

- **Веб:** CDN-версия Leaflet + MarkerCluster с загрузкой через динамическое создание `<link>` и `<script>`.
- **iOS/Android:** Leaflet внутри `WebView` — весь HTML/JS генерируется функцией `generateHTML()`.
- OSRM API: `router.project-osrm.org` — реальные дорожные маршруты между координатами. При ошибке — Catmull-Rom сплайн.
- API-ключи Google Maps в `app.json` пока не используются (заполнены плейсхолдерами).

### Управление состоянием

- **ThemeContext**: React Context + `useState` + `localStorage`.
- **UserContext**: React Context + `useReducer`. Действия: `LOGIN_AS_TEST`, `LOGOUT`, `COMPLETE_ROUTE`, `ADD_PHOTO`, `TOGGLE_WISHLIST`.
- Ачивки проверяются автоматически в редьюсере при завершении маршрута.

### Фотографии

- Два режима: `require()` для локальных файлов и `{ uri: string }` для URL.
- У каждого маршрута до 5 фото (Unsplash + локальные).
- **Система сбора:** авторизованный пользователь нажимает кнопку с камерой и «собирает» фото. Счётчик `photosCollected` в `RouteProgress` увеличивается. Несобранные фото отображаются серыми, собранные — с золотой галочкой.
- **RouteGallery** (карусель на странице маршрута): кнопка сбора фото, значок собранных (золотая камера + число) и счётчик `N/M`.
- **PhotoGallery** (профиль): сетка 3 колонки с группировкой по маршрутам, прогресс-бар общей коллекции, кнопка «Собрать фото маршрута» (пунктирная рамка), полноэкранный просмотр с кнопкой сбора.
- Достижение «Фотоохотник» (5 фото) и «Фотомастер» (15 фото) — автоматически проверяются в `UserContext`.
- Локальные фото в папках `photos/` (пользовательские) и `assets/photos/` (Grelka Fest).

### Известные особенности

- Аутентификация по телефону — симуляция (нет реального бэкенда).
- Погода — демо-режим (API-ключ `demo`, возвращает заглушку).
- Нет базы данных — все данные статичны (`src/data/`).
- Google Maps API-ключи в `app.json` — плейсхолдеры.
- Карта на iOS/Android — через WebView (react-native-maps подключён, но не используется).
